**MUST implement:**

  `junitxml` and `rerun_command` required per suite (no conditional defaults)

  via `exec.Command(command[0], command[1:]...)` — unmodified, no shell

  no suites → exit 2 with guidance; invalid name → error and abort

  CAS per file; created on first run (not during init)
- Per-suite PR comment marker `<!-- quarantine:<suite-name> -->`

- Command crash detection: non-zero exit + no JUnit XML → exit 2 + diagnostic

  genuine failures still produce exit 1 (priority rule)

  `--exclude`, `BuildExclusionArgs`, `Framework` type, exclusion logic,

## Acceptance criteria

From [index.md](index.md#m9-core-conversion):

1. `.quarantine/config.yml` parsed with `test_suites` array; `junitxml` and
   `rerun_command` required per suite; missing fields → exit 2 with
   diagnostic. (FR-1.4.1, ADR-010)
2. `quarantine run [suite-name]` executes the suite's `command` unmodified via
   `exec.Command`; no `--` separator; command never appended to. (FR-1.1.1,
   ADR-031)
3. Single suite configured + no name arg → runs the single suite. (FR-1.1.1)
4. Multiple suites + no name arg → exit 2 with list of available suite names.
   (FR-1.4.3)
5. Per-suite state file created at `.quarantine/<suite>/state.json` on first
   run; subsequent runs update it via CAS. (FR-1.2.1, ADR-032)
6. PR comment for each suite uses `<!-- quarantine:<suite-name> -->` marker.
   (FR-1.3.2, ADR-032)
7. Issue dedup label is `quarantine:<suite-name>:<hash>`; same test in two
   suites creates two separate issues. (FR-1.3.1, FR-1.3.3, ADR-032)
8. `quarantine init` detects frameworks, pre-fills suite entries, creates
   config + .gitignore + state branch; re-running is idempotent. (FR-1.4.1)
9. `quarantine doctor` validates suite names, required fields, non-empty
   `test_suites`; warns on framework retry config. (FR-1.4.3)
10. Command crash (non-zero exit + no JUnit XML) → exit 2 with
    `Error [crash]: ...` diagnostic. (ADR-031, NFR-2.2.1)
11. Rerun failure → test classified `"unresolved"`, run continues, exit 2;
    genuine failures still exit 1. (ADR-031)
12. Old format artifacts removed: `quarantine.yml`, `--config`, `--`,
    `--output`, `--exclude`, `BuildExclusionArgs`, `Framework`. (ADR-010)
13. `make cli-build && make cli-test && make cli-lint` pass.

## Verification

**Build:** `make cli-build && make cli-test && make cli-lint` MUST pass.

**Config:** The parser MUST reject `command: "npm test"` (string) with
`Error [config]: test_suites[0].command must be a YAML array, not a string.`
The parser MUST reject an empty `test_suites` array.

**Execution model:** `quarantine run` MUST call `exec.Command(command[0],
command[1:]...)` without modifying `command`. MUST NOT use `sh -c`.
MUST NOT append any flags to the command.

**Per-suite isolation:** Parallel invocations of `quarantine run backend`
and `quarantine run frontend` MUST operate on separate state files
(`.quarantine/backend/state.json` and `.quarantine/frontend/state.json`)
with no CAS contention between them (ADR-032).

**Exit code correctness:** When a run has both genuine failures and unresolved
tests, MUST exit 1 (not 2). When only unresolved (no genuine failures),
MUST exit 2.

**Requirements:** Implementation MUST satisfy [FR-1.1.1], [FR-1.2.1],
[FR-1.3.1], [FR-1.3.2], [FR-1.4.1], [FR-1.4.3] in
[functional-requirements.md](../specs/functional-requirements.md)
and [NFR-2.2.1] in
[non-functional-requirements.md](../specs/non-functional-requirements.md).
