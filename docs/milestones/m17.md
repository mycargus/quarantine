## Scope

**MUST implement:**
- Four-step token resolution: `QUARANTINE_GITHUB_TOKEN` → dashboard proxy → `GITHUB_TOKEN` → exit 2 (`cli/internal/github/`)
- `QUARANTINE_DASHBOARD_URL`: HTTPS required; localhost exception only when `CI` is unset; env-var only (no config file option)
- `QUARANTINE_DASHBOARD_TIMEOUT`: 3s default and minimum; values below minimum clamped with notice
- Fork PR detection via `GITHUB_EVENT_PATH`: head repo ≠ base repo → skip quarantine, exec raw command
- Event payload parsing hardening: malformed/missing `GITHUB_EVENT_PATH` → skip detection, quarantine runs normally
- Differentiated error messages per token resolution step (UX-2) + GHA annotations for 403 (UX-1) and proxy failure (UX-3)
- `quarantine doctor` hints `QUARANTINE_DASHBOARD_URL` upgrade path when rate limit is low (UX-8)
- Spec updates: `error-handling.md` (exit 2 replaces degraded mode for no-token; Category 4 proxy errors; proxy-minted 401 message); `cli-spec.md` (v2 token resolution order)
- Docs: CI integration guide with `pull_request_target` warning; quick-start onboarding sequence (OPS-2, UX-7)

**MUST NOT implement:**
- Dashboard proxy endpoint or token minting (M18)
- E2E round-trip testing (M19)
- GHES rate limit note, `actions:read` note (deferred per plan)

## Acceptance criteria

From [plan: adr-036-token-proxy.md#m17-cli-v2-token-resolution](../plans/adr-036-token-proxy.md#m17-cli-v2-token-resolution):

1. `GITHUB_TOKEN` is used by default when no other token is configured (R-1, FR-1.11.1 revised)
2. Token resolution follows four-step order and stops at first success (R-2)
3. `QUARANTINE_DASHBOARD_URL` must use HTTPS; rejected otherwise (R-6, NFR-2.3.x)
4. Localhost HTTPS exception allowed only when `CI` is not set (SEC-5, NFR-2.3.x)
5. `QUARANTINE_DASHBOARD_TIMEOUT` defaults to 3s, minimum 3s; values below clamped with notice (R-5, UX-5, NFR-2.1.1)
6. Dashboard URL accepted only from `QUARANTINE_DASHBOARD_URL` env var — no config file option (R-7)
7. Fork PR detected → quarantine skipped, raw test command executed (SEC-7)
8. Malformed/missing `GITHUB_EVENT_PATH` → fork detection skipped, quarantine runs normally (SEC-9)
9. 403 on write → actionable "add permissions: block" error with SSO/IP fallback hint + GHA `::error` annotation (UX-1)
10. Each token resolution failure path produces a distinct, actionable error message (UX-2, NFR-2.11.4)
11. Proxy failure warning includes URL, reason, fallback rate limit + GHA `::warning` annotation (UX-3)
12. No token available → exit 2 (not degraded mode) (ERR-1)
13. `error-handling.md` updated: degraded mode trigger 3 → exit 2; Category 4 added; proxy-minted 401 distinct (ERR-1, ERR-2, ERR-3)
14. `quarantine doctor` mentions `QUARANTINE_DASHBOARD_URL` upgrade path on low rate limit (UX-8)
15. CI docs updated with `pull_request_target` warning and full onboarding sequence (OPS-2, UX-7 — doc review)

## Verification

**Token resolution order:** The implementation MUST resolve tokens in this exact sequence:
1. `QUARANTINE_GITHUB_TOKEN` set → use it, skip proxy and `GITHUB_TOKEN`
2. `QUARANTINE_DASHBOARD_URL` set → attempt proxy call with ≤3s timeout → use returned token
3. Proxy failed or absent → use `GITHUB_TOKEN` if set
4. No token found → exit 2 with context-aware message (GHA vs non-GHA)

**Security invariants:**
- `QUARANTINE_DASHBOARD_URL` MUST be rejected if not HTTPS, EXCEPT localhost when `CI` env is unset
- Fork PR (`head.repo.full_name` ≠ `GITHUB_REPOSITORY`) MUST skip quarantine and exec raw command
- Proxy failure MUST NOT break the build — MUST fall back to `GITHUB_TOKEN`

**Requirements:** The implementation MUST satisfy FR-1.11.1 (revised) in [functional-requirements.md](../specs/functional-requirements.md) and NFR-2.11.4, NFR-2.1.1 in [non-functional-requirements.md](../specs/non-functional-requirements.md).

**Build:** `make cli-build && make cli-test && make cli-lint` MUST pass.
