# M5: GitHub Issues + PR Comments + Result Artifacts

## Workflow

1. Read this manifest to understand scope and constraints.
2. Load the [acceptance scenarios](#acceptance-scenarios) for the current command.
3. Use `/mikey:tdd <scenario-file>` to implement against those scenarios.
4. After implementation, verify against [acceptance criteria](#acceptance-criteria)
   and [flow verification](#verification).
5. Use `/mikey:testify` to validate test quality.

## Scope

**MUST implement:**
- GitHub Issue creation for newly detected flaky tests with dedup via
  Search API and deterministic `quarantine:{test_hash}` label
- PR comment posting/updating identified by `<!-- quarantine-bot -->` marker
- `--pr N` flag override; auto-detect from `GITHUB_EVENT_PATH`
- Comment template per cli-spec with conditional sections
- Skipped when `notifications.github_pr_comment: false` or no PR number
- Complete result JSON with all metadata for dashboard ingestion
- Error handling: dedup failure → create anyway; comment failure → best-effort;
  issue failure → log warning, write state without `issue_number`

**MUST NOT implement:**
- Artifacts REST API upload (workflow handles via `actions/upload-artifact`)
- Dashboard integration
- Any new state management (uses M4's implementation)

## Acceptance criteria

From [milestones.md](../planning/milestones.md#m5-github-issues--pr-comments--result-artifacts):

1. New flaky test triggers issue creation with correct title, body, labels.
   (FR-1.3.1)
2. Dedup prevents duplicate issues for the same test. (FR-1.3.3)
3. PR comment posted on triggering PR with quarantine summary. (FR-1.3.2,
   FR-1.12.1)
4. Second run on same PR updates existing comment (no duplicates).
5. `--pr` flag overrides auto-detected PR number.
6. `notifications.github_pr_comment: false` suppresses PR comment.
7. `.quarantine/results.json` contains complete dashboard metadata.
8. **Stakeholder demo:** quarantined test excluded, flaky test triggers issue
   and PR comment, results written.
9. Unit tests cover: issue/comment templates, dedup query, `test_hash`,
   PR number detection from `GITHUB_EVENT_PATH`.
10. E2E test: full flow against a real GitHub repo and PR. (see `e2e/run.test.js`)

[non-functional](../planning/non-functional-requirements.md).

## Verification

**Flow correctness:** The implementation MUST match
[sequence-diagrams.md#1](../specs/sequence-diagrams.md#1-happy-path-flaky-test-detected-and-quarantined).
Specifically:
- Issue dedup MUST search before creating (check-before-create)
- PR comment search MUST be capped at 100 most recent comments
- Comment marker `<!-- quarantine-bot -->` MUST be on the first line
- 410 Gone MUST skip issue creation for ALL flaky tests in the run
- Issue body MUST include failure message and retry results
- `GITHUB_EVENT_PATH` parsing MUST handle missing file or missing
  `pull_request` field gracefully

**Requirements:** The implementation MUST satisfy FR-1.3.1, FR-1.3.2,
FR-1.3.3, FR-1.3.4, FR-1.12.1 in
[functional-requirements.md](../planning/functional-requirements.md).

**Build:** `make cli-build && make cli-test && make cli-lint` MUST pass.
