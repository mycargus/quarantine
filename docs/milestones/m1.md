# M1: CLI Scaffolding + `quarantine init`

## Workflow

1. Read this manifest to understand scope and constraints.
2. Load the [acceptance scenarios](#acceptance-scenarios) for the current command.
3. Use `/mikey:tdd <scenario-file>` to implement against those scenarios.
4. After implementation, verify against [acceptance criteria](#acceptance-criteria)
   and [flow verification](#verification).
5. Use `/mikey:testify` to validate test quality.

## Scope

**MUST implement:**
- `quarantine init` — interactive setup command
- `quarantine doctor` — config validation command
- Config parsing and validation (`internal/config/`)
- GitHub client foundation (`internal/github/`)
- Git remote parsing for owner/repo auto-detection (`internal/git/`)

**MUST NOT implement:**
- `quarantine run`, JUnit XML parsing, retry logic, quarantine state
  read/write beyond initial creation, GitHub Issues, PR comments, artifact
  upload, `--yes` flag for non-interactive init
- Degraded mode in `init` — all failures MUST be fatal (exit 2) with
  actionable diagnostics

**`quarantine version`** is already implemented.

## Acceptance criteria

From [milestones.md](../planning/milestones.md#m1-cli-scaffolding-quarantine-init):

1. `quarantine init` creates a valid `quarantine.yml` and
   `quarantine/state` branch with empty `quarantine.json`. (FR-1.4.1, FR-1.11.1)
2. `quarantine doctor` reports errors for invalid config and prints resolved
   config for valid config. (FR-1.4.3)
3. `quarantine version` prints the version string.
4. `make cli-build` produces a binary. `make cli-test` and `make cli-lint` pass. (NFR-2.5.1)
5. Unit tests cover: config parsing/validation (all fields, all error cases),
   token resolution, git remote parsing. (FR-1.4.2)
6. E2E test: `quarantine init` against a real GitHub repo creates the branch
   and file. (see `test/e2e/init.test.js`)

Requirements: [functional](../planning/functional-requirements.md),
[non-functional](../planning/non-functional-requirements.md).

## Verification

**Flow correctness:** The implementation MUST match the sequence in
[sequence-diagrams.md — 8. quarantine init](../specs/sequence-diagrams.md#8-quarantine-init).
Specifically:
- `init` MUST call endpoints in this order: `GET /repos` → `GET /git/ref` →
  `POST /git/refs` → `PUT /contents`
- `init` MUST exit 2 on any GitHub API failure (no degraded mode)
- `init` MUST skip branch creation if branch already exists
- `init` MUST write config before attempting GitHub operations

**Requirements:** The implementation MUST satisfy FR-1.4.1, FR-1.4.2,
FR-1.4.3, FR-1.11.1 in [functional-requirements.md](../planning/functional-requirements.md)
and NFR-2.5.1 in [non-functional-requirements.md](../planning/non-functional-requirements.md).

**Build:** `make cli-build && make cli-test && make cli-lint` MUST pass.
