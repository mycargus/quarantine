# M2: Test Execution + JUnit XML Parsing

## Workflow

1. Read this manifest to understand scope and constraints.
2. Load the [acceptance scenarios](#acceptance-scenarios) for the current command.
3. Use `/mikey:tdd <scenario-file>` to implement against those scenarios.
4. After implementation, verify against [acceptance criteria](#acceptance-criteria)
   and [flow verification](#verification).
5. Use `/mikey:testify` to validate test quality.

## Scope

**MUST implement:**
- `quarantine run -- <command>` — test execution wrapper
- JUnit XML parser with Jest, RSpec, Vitest support
- `test_id` construction as `file_path::classname::name`
- Result JSON output to `.quarantine/results.json`
- Glob resolution for `junitxml` pattern (single and multiple files)
- Signal handling: forward SIGINT/SIGTERM to child process
- `--config`, `--junitxml`, `--verbose`, `--quiet` flags
- Prerequisite check: verify `quarantine/state` branch exists
- Golden test fixtures in `testdata/` for all three frameworks

**MUST NOT implement:**
- Retry logic, `--retries` flag
- Quarantine state read/write beyond the branch existence check
- GitHub Issues, PR comments, state updates
- `--dry-run`, `--strict`, `--pr`, `--exclude` flags

## Acceptance criteria

From [milestones.md](../planning/milestones.md#m2-test-execution--junit-xml-parsing):

1. `quarantine run -- jest --ci` executes Jest, parses JUnit XML, and writes
   `results.json` with correct test counts. (FR-1.1.1, FR-1.1.2)
2. `quarantine run -- rspec` executes RSpec and parses its XML output.
3. `quarantine run -- vitest run` executes Vitest and parses its XML output.
4. Exit 0 when all tests pass. Exit 1 when any test fails.
5. Exit 2 when `quarantine init` has not been run.
6. Glob patterns for `junitxml` resolve multiple files and merge results.
7. Malformed XML produces a warning and the CLI exits with the runner's code.
8. Unit tests cover: JUnit XML parsing for all three frameworks (golden
   fixtures), test_id construction, result JSON serialization. (FR-1.1.5)

Requirements: [functional](../planning/functional-requirements.md),
[non-functional](../planning/non-functional-requirements.md).

## Verification

**Flow correctness:** The implementation MUST match
[sequence-diagrams.md#1](../specs/sequence-diagrams.md#1-happy-path-flaky-test-detected-and-quarantined)
(run + parse portion only). Specifically:
- The `--` separator MUST be required. Missing separator MUST exit 2.
- Branch check via `GET /repos/{owner}/{repo}/git/ref/heads/quarantine/state`.
  404 MUST exit 2 with "Run 'quarantine init' first."
- JUnit XML glob resolution MUST handle zero, one, or multiple files.
- `file_path` extraction MUST use framework-specific rules from cli-spec.
- Result JSON MUST include metadata: commit SHA, branch, timestamp, CLI version.

**Requirements:** The implementation MUST satisfy FR-1.1.1, FR-1.1.2,
FR-1.1.5, FR-1.1.6 in [functional-requirements.md](../planning/functional-requirements.md)
and NFR-2.1.1 in [non-functional-requirements.md](../planning/non-functional-requirements.md).

**Build:** `make cli-build && make cli-test && make cli-lint` MUST pass.
