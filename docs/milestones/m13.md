## Scope

**MUST implement:**
- Add `@remix-run/auth`, `@remix-run/auth-middleware`, `@remix-run/session`, `@remix-run/session-middleware` to `dashboard/package.json` with exact version pins (not `^` ranges) since all are pre-1.0 (ADR-029)
- GitHub OAuth provider via `createGitHubAuthProvider({ clientId, clientSecret, redirectUri })` from `@remix-run/auth` â†’ `dashboard/app/lib/auth.server.ts`
- Session management: `createCookieSessionStorage` from `@remix-run/session` with `maxAge: 28800` (8 hours, matching GitHub access token lifetime). The encrypted cookie holds the user's access token and profile. No `sessions` table, no refresh tokens, no server-side session state. When the cookie expires, the user re-authenticates via OAuth. No SQLite migration needed for sessions.
- Router-level middleware: `session()` + `auth()` with `createSessionAuthScheme`
- New routes: `/auth/login`, `/auth/github/callback`, `/auth/logout`, `/health`
- Route protection via `requireAuth()` middleware: all routes except `/auth/*` and `/health` MUST return 401 when unauthenticated
- Rate limiting in two layers via custom `remix/fetch-router` middleware (the router supports middleware chaining via `Middleware` types, but does not ship a built-in rate limiter â€” application must implement the rate limiting logic): (1) IP-based limit (20 req/min/IP) as middleware before auth resolution â€” applies to all requests including unauthenticated; (2) user-based limit (300 req/min/user) as middleware after `session()` + `auth()` â€” applies only to authenticated requests. Both layers use **fixed-window** counters (reset every 60 seconds from the start of each window). `Retry-After` header value = seconds remaining until the current window resets. Both layers respond with HTTP 429 + `Retry-After` header (seconds) per RFC 6585 / RFC 9110. IP extraction: read client IP from `X-Forwarded-For` header (first entry) when behind a reverse proxy, falling back to the socket remote address. The trusted header is configurable to support different proxy setups. Rate limit window tracking MUST accept an injectable clock function (default `Date.now`) for testability per FC/IS principle â€” 60-second windows are impractical to test in real time.
- Auth event logging: timestamp + user ID; MUST NOT log token values
- Interface tests â†’ `dashboard/test/oauth-flow.integration.test.ts`

**Required environment variables (dashboard reads these at runtime):**
- `QUARANTINE_APP_CLIENT_ID` â€” OAuth client ID (same as JWT `iss` in M12)
- `QUARANTINE_APP_CLIENT_SECRET` â€” OAuth client secret
- `QUARANTINE_APP_ORIGIN` â€” dashboard origin for constructing `redirectUri` (e.g., `http://localhost:3000` for dev, `https://dashboard.example.com` for prod)

**MUST NOT implement:**
- User permission filtering via `GET /user/installations` (M15)
- Installation discovery, `installations` table, or `source: github-app` config (M14)
- JWT generation or `InstallationTokenProvider` (M12)
- E2E OAuth tests requiring browser interaction with GitHub consent page (M16)

**Operational prerequisite:** The dev GitHub App must be registered (M16 one-time setup step 1) before local development or manual testing. Interface tests use mock servers and do not require real App credentials.

## Acceptance criteria

From [test-strategy.md](../specs/test-strategy.md) and [ADR-029](../adr/029-dashboard-oauth-first-party.md):

1. `createGitHubAuthProvider` from `@remix-run/auth` configured with `clientId` (`QUARANTINE_APP_CLIENT_ID`), `clientSecret` (`QUARANTINE_APP_CLIENT_SECRET`), and `redirectUri` (constructed from `QUARANTINE_APP_ORIGIN` + `/auth/github/callback`). (FR-1.15.1, ADR-029)
2. Authenticated `GET /` â†’ 200. Unauthenticated `GET /` â†’ 401. (FR-1.15.7)
3. `GET /auth/login` calls `startExternalAuth(githubProvider, context)`, redirecting to GitHub OAuth authorization URL with PKCE. (FR-1.15.8)
4. `GET /auth/logout` clears the session and redirects to `/auth/login`. (FR-1.15.8)
5. `GET /health` â†’ 200 without authentication, always. (FR-1.15.7, FR-1.15.8)
6. Session cookie MUST have `httpOnly`, `secure`, `SameSite=Lax` attributes, and `Max-Age: 28800` (8 hours). The encrypted cookie holds the access token and user profile â€” no server-side session table. When the cookie expires, the user re-authenticates via OAuth. (FR-1.15.3)
7. Refresh tokens are NOT stored or used. The session lifetime matches the GitHub access token lifetime (8 hours). After expiry, the user re-authenticates. This eliminates refresh token rotation handling entirely. **Prerequisite:** the dev GitHub App MUST have "User-to-server token expiration" enabled (M16 one-time setup step 1). Without this opt-in setting, GitHub access tokens do not expire and the "session matches token lifetime" invariant does not hold. (FR-1.15.2)
8. Rate limiting enforced: unauthenticated 20 req/min/IP; authenticated 300 req/min/user. Rate-limited responses MUST return HTTP 429 with `Retry-After` header (seconds). (NFR-2.3.3)
9. Auth events logged with timestamp and user ID. Token values MUST NOT appear in any log output. (NFR-2.7.5)
10. `make dash-test` MUST pass.

## Verification

**Build:** `make dash-test` MUST pass.

**Route protection (per [FR-1.15.7](../specs/functional-requirements.md#115-dashboard-oauth), [diagram 11](../specs/sequence-diagrams.md#11-dashboard-oauth-login-flow-v2)):**
- Unauthenticated request to any route except `/auth/login`, `/auth/github/callback`,
  `/auth/logout`, and `/health` MUST return 401.
- `GET /health` MUST return 200 regardless of authentication state.
- `GET /auth/logout` with an expired or absent session MUST NOT return an error â€” it redirects to `/auth/login` (clearing the cookie is a no-op when the session is already expired).

**Session invariants:**
- Session cookie MUST carry `httpOnly`, `secure`, `SameSite=Lax` flags, and `Max-Age: 28800`.
- After `GET /auth/logout`, a repeat request MUST return 401.
- After 8 hours (cookie expiry), a request MUST return 401 (user re-authenticates via OAuth).
- No refresh tokens are stored or used. No `sessions` table exists.

**Rate limit invariants:**
- IP-based layer MUST extract client IP from a configurable header (default `X-Forwarded-For`, first entry) with fallback to socket remote address. Behind a reverse proxy (NFR-2.3.1), all requests share the proxy IP unless the real client IP is extracted.
- Rate limit counters MUST accept an injectable clock function (default `Date.now`) so interface tests can advance time without waiting 60 real seconds.

**Logging invariant (per [NFR-2.7.5](../specs/non-functional-requirements.md#27-app-auth-security)):**
- Log entries for login and logout MUST include timestamp and user ID.
- Token values (`ghu_` access tokens) MUST NOT appear in any log output.
