## Scope

**MUST implement:**
- `POST /api/ci-token` endpoint (`dashboard/app/routes/api.ci-token.ts`): two-phase verification (soft prefix gate → behavioral check via hardcoded GitHub API URL)
- Token hygiene: received `GITHUB_TOKEN` never logged, never persisted, discarded after verification (SEC-1)
- Verification cache: keyed by `SHA-256(token)[0:16]:owner/repo`, 15s TTL, expired entries evicted (SEC-2)
- Scoped installation token minting: `repositories: ["<repo>"]` parameter required — MUST NOT reuse existing `InstallationTokenProvider` without modification (SEC-6a)
- Installation token cache: 1hr TTL, proactive refresh when <5min remaining (R-3)
- Rate limiting: 10/min per repo + 10/min per token fingerprint; 429 with `Retry-After`; evict expired counters (SEC-6)
- SSRF protection: behavioral check target URL MUST be hardcoded GitHub API base, never derived from request input (SEC-6b)
- App permission verification at dashboard startup: log warning for any permission beyond SEC-3 table (OPS-1)
- Revocation procedure documented in operational runbook (SEC-8 — doc review)

**MUST NOT implement:**
- CLI token resolution changes (M17)
- E2E round-trip testing (M19)
- Reuse of existing `InstallationTokenProvider` without adding `repositories` scope parameter

## Acceptance criteria

From [plan: adr-036-token-proxy.md#m18-dashboard-token-proxy](../plans/adr-036-token-proxy.md#m18-dashboard-token-proxy):

1. `POST /api/ci-token` accepts tokens and returns scoped installation tokens (R-3)
2. Known-bad prefixes (`ghp_`, `github_pat_`, `ghu_`, `gho_`) rejected with 404 immediately; unknown prefixes fall through to behavioral check (R-4)
3. Behavioral check calls `GET /installation/repositories` on hardcoded GitHub API base URL (R-4, SEC-6b)
4. Received `GITHUB_TOKEN` never logged, never persisted, discarded after verification (SEC-1, NFR-2.7.x)
5. Verification cache keyed by `SHA-256(token)[0:16]:owner/repo`, 15s TTL, expired entries evicted (SEC-2, NFR-2.8.x)
6. Installation token minting passes `repositories: ["<repo>"]` to scope token to single repo (SEC-6a)
7. Installation token cache: 1hr TTL, proactive refresh when <5min remaining (R-3, NFR-2.8.1)
8. Rate limiting: 10/min per repo + 10/min per token fingerprint; 429 with `Retry-After` on excess (SEC-6, NFR-2.3.3)
9. Rate limit counters evict expired entries to prevent memory leak (SEC-6)
10. App permissions beyond SEC-3 table logged as warning at startup; exact match produces no warning (OPS-1)
11. 404 returned for repos not in any installation — no information leakage (R-3)
12. Revocation procedure documented in operational runbook (SEC-8 — doc review)
13. Unknown-prefix tokens that fail behavioral check: 404 (R-4)

Requirements: [functional](../specs/functional-requirements.md), [non-functional](../specs/non-functional-requirements.md).

## Verification

**Two-phase verification order:** The implementation MUST follow:
1. Check token prefix — known-bad (`ghp_`, `github_pat_`, `ghu_`, `gho_`) → 404 immediately
2. Unknown prefix → behavioral check via `GET /installation/repositories` on hardcoded GitHub API URL
3. Token verified → check verification cache (15s TTL); on miss, call GitHub API; on hit, skip API call
4. Mint installation token with `repositories: ["<repo>"]` scope; cache with 1hr TTL

**Security invariants:**
- Behavioral check target URL MUST be hardcoded — MUST NOT use any value from the request
- Received token MUST NOT appear in any log output or be written to any storage
- Tokens for repos not in the installation MUST return 404 — no info leakage
- Rate limit Maps MUST evict expired entries to prevent unbounded growth

**Requirements:** The implementation MUST satisfy NFR-2.7.x in [non-functional-requirements.md](../specs/non-functional-requirements.md#27-app-auth-security) and NFR-2.8.x in [non-functional-requirements.md](../specs/non-functional-requirements.md#28-app-auth-performance).

**Build:** `make dash-test && make dash-lint && make dash-typecheck` MUST pass.
