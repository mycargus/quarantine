# M8: Polish and Hardening

## Workflow

1. Read this manifest to understand scope and constraints.
2. Load the [acceptance scenarios](#acceptance-scenarios) for the current command.
3. Use `/mikey:tdd <scenario-file>` to implement against those scenarios.
4. After implementation, verify against [acceptance criteria](#acceptance-criteria)
   and [flow verification](#verification).
5. Use `/mikey:testify` to validate test quality.

## Scope

**MUST implement:**
- Error handling testing: every error path from `error-handling.md` MUST have
  a corresponding test (CLI and dashboard)
- Degraded mode testing: simulate GitHub API failures across all scenarios
- CLI Docker image: minimal image with Go binary
- Dashboard Docker image: Node.js image with built app, SQLite, volume mount
- README.md: project overview, quick start, CI integration guide, workflow
  examples, troubleshooting
- Parameterized test fixtures: Jest `test.each`, RSpec shared examples,
  Vitest parameterized — verify `test_id` construction
- CLI cross-compilation: linux/darwin/windows x amd64/arm64 (6 targets)
- Release automation: GitHub Actions workflow for build, SHA-256 checksums,
  publish

**MUST NOT implement:**
- New features — M8 is exclusively polish, hardening, and documentation
- v2 features (GitHub App, Jira, Slack, auto-fix, etc.)

## Acceptance criteria

From [milestones.md](../planning/milestones.md#m8-polish-and-hardening):

1. Every error path in `error-handling.md` has at least one test.
2. Degraded mode scenarios pass: no state, API timeout, CAS exhaustion,
   token expired.
3. CLI Docker image runs `quarantine version` successfully. (NFR-2.5.3)
4. Dashboard Docker image starts and serves the UI. (NFR-2.5.4)
5. README: quick start < 5 min, CI integration for all frameworks,
   troubleshooting for common errors.
6. Parameterized test fixtures produce valid `test_id` values.
7. Cross-compiled binaries run on target platforms. (NFR-2.5.1)
8. `make test-all` passes (cli-test + dash-test + schemas-validate).
9. v1 feature-complete per architecture.md §8 roadmap.

Requirements: [functional](../planning/functional-requirements.md),
[non-functional](../planning/non-functional-requirements.md).

## Acceptance scenarios

M8 has no dedicated scenarios. It validates that ALL prior milestone
scenarios pass end-to-end, plus:

- Error handling paths not covered by scenario-level tests
- Parameterized test fixture parsing (new golden fixtures in `testdata/`)
- Cross-platform binary execution

## Verification

**Flow correctness:** No new flows. Verification is coverage-focused:
- Read [error-handling.md](../specs/error-handling.md) section by section;
  for each error path, a test MUST exist
- Degraded mode matrix: 401, 403, 404, 409, 422, 429, 500, 503, timeout ×
  (cache hit, cache miss) × (`--strict`, default)
- Quick start guide MUST be completable in < 5 min by a new user
- CI examples MUST be copy-pasteable for Jest, RSpec, Vitest
- Troubleshooting MUST cover: missing token, missing XML, branch not found,
  rate limit exceeded

**Requirements:** The implementation MUST satisfy NFR-2.5.1, NFR-2.5.2,
NFR-2.5.3, NFR-2.5.4 in
[non-functional-requirements.md](../planning/non-functional-requirements.md).

**Build:** `make cli-build && make cli-test && make cli-lint && make dash-build && make dash-test && make dash-lint` MUST pass.
