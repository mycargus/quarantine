# M7: Dashboard Analytics and UI

## Workflow

1. Read this manifest to understand scope and constraints.
2. Load the [acceptance scenarios](#acceptance-scenarios) for the current command.
3. Use `/mikey:tdd <scenario-file>` to implement against those scenarios.
4. After implementation, verify against [acceptance criteria](#acceptance-criteria)
   and [flow verification](#verification).
5. Use `/mikey:testify` to validate test quality.

## Scope

**MUST implement:**
- Quarantined test list per project: test name, issue link, quarantine date,
  flaky count
- Trend charts: flaky test count over time, test stability trend (pass rate),
  quarantine queue size over time
- Filters: by project, date range, test status
- Search: by test name or `test_id`
- Cross-repo overview page: aggregated stats (total quarantined, total flaky
  detections, most flaky tests)
- Integration tested with real CLI output from M5
- Responsive layout with Tailwind CSS

**MUST NOT implement:**
- Manual quarantine/unquarantine from UI (read-only in v1)
- Docker image (M8)
- Authentication beyond network-level
- Export or API endpoints

## Acceptance criteria

From [milestones.md](../planning/milestones.md#m7-dashboard-analytics-and-ui):

1. Project detail page shows quarantined tests with issue links. (FR-1.5.1)
2. Trend charts render correctly with at least 7 days of data.
3. Filters narrow the displayed data correctly.
4. Search returns matching tests by name or `test_id`.
5. Cross-repo overview page shows aggregated statistics. (FR-1.5.5)
6. Dashboard ingests and displays data from real CLI output (not just fixtures).
7. UI is responsive on desktop and tablet viewports.
8. Unit and integration tests cover: chart data queries, filter logic, search,
   cross-repo aggregation.

Requirements: [functional](../planning/functional-requirements.md),
[non-functional](../planning/non-functional-requirements.md).

## Verification

**Flow correctness:** No sequence diagram applies. Verification is
data-correctness focused:
- Trend chart queries MUST aggregate by date, one data point per day
- Cross-repo aggregation MUST sum across all repos in `dashboard.yml`
- Search MUST be case-insensitive and match partial test names
- Filters MUST compose (project + date range + status applied together)
- Quarantine test list MUST link to the correct GitHub Issue URL
- MUST validate that real CLI output (from M5) ingests and displays correctly
- Trend charts MUST handle sparse data (days with no runs)

**Requirements:** The implementation MUST satisfy FR-1.5.1, FR-1.5.5 in
[functional-requirements.md](../planning/functional-requirements.md) and
NFR-2.1.2 in
[non-functional-requirements.md](../planning/non-functional-requirements.md).

**Build:** `make dash-build && make dash-test && make dash-lint` MUST pass.
