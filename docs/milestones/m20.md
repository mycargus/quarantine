# M20: Decouple Git Origin From GitHub Target — Jenkins / Non-GitHub-Origin Support

## Workflow

1. Read this manifest to understand scope and constraints.
2. Load the [acceptance scenarios](#acceptance-scenarios) for the current command.
3. Use `/mikey:tdd <scenario-file>` to implement against those scenarios.
4. After implementation, verify against [acceptance criteria](#acceptance-criteria)
   and [flow verification](#verification).
5. Use `/mikey:testify` to validate test quality.

## Scope

**MUST implement:**
- Remove `u.Host == "github.com"` hard check in `cli/internal/git/remote.go:36` and remove git-remote scanning from `run` and `doctor` owner/repo resolution paths
- `quarantine init` two-phase flow: phase 1 writes `.quarantine/config.yml` with empty `github` block (commented placeholders) and exits 2; phase 2 (re-run after hand-edit) validates token, creates `quarantine/state` branch, exits 0
- `quarantine init` phase 1 best-effort hint: inspect `git remote -v`; if any remotes are github.com URLs, append YAML comments under the `github` block listing each as `# <remote-name> -> <owner>/<repo>`. If `git remote -v` fails or no github.com remotes exist, write the empty block with no hints. Hints are advisory only and ignored by all other code paths
- Owner/repo resolution: read from `.quarantine/config.yml` only; missing or empty `github.owner`/`github.repo` produces `Error [config]:` and exit 2 in `init` (phase 2), `run`, and `doctor`
- Config schema update: `github.owner` and `github.repo` are required (non-empty strings)
- `quarantine doctor` MUST read owner/repo from config (not from git origin), make a single `GET /repos/{owner}/{repo}` reachability call, exit 0 on 200, exit 2 on 4xx with GitHub's status surfaced
- `quarantine doctor` MUST NOT inspect git origin URL, `response.permissions`, or `response.has_issues` (token-scope and feature-flag introspection are out of scope)
- Interface tests: Jenkins env (no `GITHUB_*` vars, config-provided owner/repo), `--pr N` path, no-token startup error, missing-config error, doctor 404 reachability failure
- Contract test: `GET /repos/{owner}/{repo}` request/response shape used by doctor (against vendored OpenAPI extract)
- `docs/guides/jenkins-integration.md`: token setup, Jenkinsfile snippet, two-phase init walkthrough, `--pr` flag, limitations (no dashboard ingestion, no auto-provisioned token, SHA provenance caveat)
- Amend project-root `CLAUDE.md` "Boundaries" section: replace the line `**v1 CI:** GitHub Actions only for full features` with `**v1 CI:** Full features (PR comments, dashboard ingestion) require GitHub Actions; core CLI (state CAS, issue create/dedup, JUnit parsing, retries, results.json) works on any CI runner with `QUARANTINE_GITHUB_TOKEN` and network access to GitHub`
- Mark Scenario 9 in `docs/scenarios/v1/01-initialization.md` as superseded by M20

**MUST NOT implement:**
- Authoritative auto-discovery: hint comments must NOT pre-fill `github.owner` or `github.repo`. Resolution reads the explicit fields only
- Git-remote scanning in `run` or `doctor` (only `init` phase 1 may scan, and only to emit hint comments)
- New CLI flags (`--owner`, `--repo`) or new env vars for PR number injection
- No-op write probes in `doctor` — permission inference is accepted (see ADR-037 limitations)
- Dashboard ingestion of Jenkins artifacts or Jira ticket integration (separate ADRs)

## Acceptance criteria

From [ADR-037](../adr/037-relax-origin-host-constraint.md) and
[milestones/index.md](index.md):

1. `quarantine init` first run writes config with empty `github` block, exits 2 with hand-edit instructions. (FR-1.4.4, NFR-2.2.1)
2. `quarantine init` re-run with valid config validates token, creates `quarantine/state` branch (idempotent), exits 0. (FR-1.4.4, NFR-2.2.4)
3. `quarantine run` reads `github.owner`/`github.repo` from config without inspecting git remote. (FR-1.4.4)
4. `quarantine run` with missing `github.owner`/`github.repo` exits 2 before executing the test command. (FR-1.4.4, NFR-2.2.1)
5. `quarantine run` with no `QUARANTINE_GITHUB_TOKEN` or `GITHUB_TOKEN` exits 2 immediately (existing behavior; verify regression coverage). (NFR-2.2.1)
6. `quarantine doctor` with valid config and reachable target passes; no origin-format warning. (FR-1.4.4)
7. `quarantine doctor` with missing `github.owner`/`github.repo` exits 2 without making any GitHub API call. (FR-1.4.4)
8. `quarantine doctor` against an unreachable target (`GET /repos` returns 4xx) exits 2 surfacing GitHub's status. (FR-1.4.4)
9. `quarantine run --pr 42` posts PR comment when `GITHUB_EVENT_PATH` is unset. (FR-1.3.2)
10. `quarantine run` with no PR context silently skips PR comment; run completes normally. (FR-1.3.2)
11. `docs/guides/jenkins-integration.md` created. (NFR-2.6.1)
12. `make cli-build && make cli-test && make cli-lint` pass.

## Verification

**Flow correctness:** Owner/repo resolution MUST follow
[ADR-037 decision](../adr/037-relax-origin-host-constraint.md#decision):
- MUST read `github.owner`/`github.repo` from `.quarantine/config.yml` only
- MUST NOT inspect any git remote URL in `init`, `run`, or `doctor`
- MUST exit 2 with `Error [config]:` if owner or repo is missing or empty
- MUST exit 2 in `init` phase 1 (partial config written, hand-edit required)
- `doctor` MUST call `GET /repos/{owner}/{repo}` once, pass on 200, exit 2 on 4xx
- `doctor` MUST NOT introspect token scopes (`response.permissions`) or
  feature flags (`response.has_issues`) — token-scope problems surface at
  runtime with GitHub's own error

**Requirements:** Satisfies [FR-1.3.2, FR-1.4.4](../specs/functional-requirements.md)
and [NFR-2.2.1, NFR-2.2.4, NFR-2.6.1](../specs/non-functional-requirements.md).

**Build:** `make cli-build && make cli-test && make cli-lint` MUST pass.
