# M11: Init UX Improvements

## Workflow

1. Read this manifest to understand scope and constraints.
2. Load the [acceptance scenarios](#acceptance-scenarios) for the current command.
3. Use `/mikey:tdd <scenario-file>` to implement against those scenarios.
4. After implementation, verify against [acceptance criteria](#acceptance-criteria)
   and [flow verification](#verification).
5. Use `/mikey:testify` to validate test quality.

## Scope

**MUST implement:**
- New `cli/internal/detect` package: `Scan(dir string) Result` (never returns error)
- Detection from `package.json` (`dependencies` + `devDependencies`) and `Gemfile`
- Exported pure parsing functions: `ParsePackageJSON`, `ParseGemfile`
- `DetectedFramework{Name, Source}` type with source metadata
- Vitest-before-jest ordering when both detected (migration direction)
- `rspec-core` detection via regex (`gem\s+['"]rspec(-core)?['"]`)
- Gemfile comment handling (skip `#`-prefixed lines)
- `Result.Names()` helper for init.go integration
- Per-suite retries validation (1-10) in `ValidateSuites` via `ValidateSuiteRetries`

**MUST NOT implement:**
- `--yes`, `--framework`, `--retries`, `--junitxml` flags (deferred)
- Interactive prompts for retries or junitxml
- TTY detection or warning
- `--config` flag on `quarantine init`
- Changes to `quarantine run`, `quarantine doctor`, or `quarantine version`

**Descoped from original plan:**
- Interactive init prompts (current init is non-interactive; auto-detection
  fills defaults, user edits YAML after)
- `--yes` flag (no prompts to skip; `--force` for config reset deferred)
- `--retries` and `--junitxml` flags (marginal value; user edits YAML)
- TTY warning (no stdin reading, so irrelevant)
- ADRs for auto-detection and non-interactive init (auto-detection already
  implemented in M10; non-interactive init descoped)

## Acceptance criteria

1. `cli/internal/detect` package exists with `Scan(dir string) Result` API. (FR-1.1.6)
2. `Scan` detects jest/vitest from `package.json` (dependencies + devDependencies)
   and rspec/rspec-core from `Gemfile`, with source metadata.
3. Vitest appears before jest when both are detected.
4. Commented gems in `Gemfile` are skipped.
5. Malformed `package.json` → silent fallback (empty result, no error).
6. `ValidateSuites` rejects per-suite retries outside 1-10 range.
7. Per-suite retries of 0 (unset/default) is accepted.
8. `make cli-build && make cli-test && make cli-lint` pass.

Requirements: [functional](../specs/functional-requirements.md),
[non-functional](../specs/non-functional-requirements.md).

## Verification

**Build:** `make cli-build && make cli-test && make cli-lint` MUST pass.

**Detect package:** `detect.Scan()` MUST never return an error (advisory-only API).
Detection failures MUST silently fall back to empty result.

**Retries:** Per-suite retries in `ValidateSuites` MUST reject values outside
1-10 with a clear error. 0 (unset) MUST be accepted.

**Requirements:** Implementation MUST satisfy [FR-1.1.6](../specs/functional-requirements.md).
