# M3: Flaky Detection via Retry

## Workflow

1. Read this manifest to understand scope and constraints.
2. Load the [acceptance scenarios](#acceptance-scenarios) for the current command.
3. Use `/mikey:tdd <scenario-file>` to implement against those scenarios.
4. After implementation, verify against [acceptance criteria](#acceptance-criteria)
   and [flow verification](#verification).
5. Use `/mikey:testify` to validate test quality.

## Scope

**MUST implement:**
- Retry logic: re-run individual failing tests using framework-specific rerun
  commands (Jest, RSpec, Vitest)
- `--retries N` flag (range 1–10, overrides config; default from config or 3)
- Flaky classification: fails initially, passes on any retry → flaky
- Genuine failure classification: fails all retries → genuine failure
- Exit code: flaky tests = exit 0, genuine failures = exit 1
- `rerun_command` config override with `{name}`, `{classname}`, `{file}`
  placeholders
- Updated result JSON with retry details per test
- Early exit from retry loop on first passing attempt

**MUST NOT implement:**
- Quarantine state read/write (beyond the init check from M2)
- Test exclusions, GitHub Issues, PR comments, state updates
- `--dry-run`, `--strict`, `--pr`, `--exclude` flags

## Acceptance criteria

From [milestones.md](../planning/milestones.md#m3-flaky-detection-via-retry):

1. A flaky test (fails first, passes on retry) is classified as flaky.
   CLI exits 0. (FR-1.1.3, FR-1.1.4)
2. A genuinely failing test (fails all retries) exits 1.
3. `--retries 1` re-runs once. `--retries 5` re-runs up to 5 times.
4. Framework-specific rerun commands work for all three frameworks. (FR-1.1.6)
5. Custom `rerun_command` in config is used when set. (FR-1.1.7)
6. Result JSON includes retry details for each retried test.
7. Unit tests cover: flaky classification, rerun command construction,
   placeholder substitution, retry count bounds validation.

## Verification

**Flow correctness:** The retry implementation MUST follow
[sequence-diagrams.md#1](../specs/sequence-diagrams.md#1-happy-path-flaky-test-detected-and-quarantined)
(retry portion only). Specifically:
- Each failing test MUST be re-run individually, not as a batch
- Rerun commands MUST use the framework-specific command or the
  `rerun_command` override from config
- The retry loop MUST exit early on the first passing attempt
- Jest `--testNamePattern` MUST escape regex special characters in test names
- `--retries` flag MUST be validated: range 1–10, exit 2 if out of range

**Requirements:** The implementation MUST satisfy FR-1.1.3, FR-1.1.4,
FR-1.1.6, FR-1.1.7 in [functional-requirements.md](../planning/functional-requirements.md).

**Build:** `make cli-build && make cli-test && make cli-lint` MUST pass.
