# M10: Additive Features

## Workflow

1. Read this manifest to understand scope and constraints.
2. Load the [acceptance scenarios](#acceptance-scenarios) for the current command.
3. Use `/mikey:tdd <scenario-file>` to implement against those scenarios.
4. After implementation, verify against [acceptance criteria](#acceptance-criteria)
   and [flow verification](#verification).
5. Use `/mikey:testify` to validate test quality.

## Scope

**Before coding, run:**
- `/create-user-scenario` for all M10 scenarios listed under
  [Acceptance scenarios](#acceptance-scenarios) — scenarios do not yet exist.

**MUST implement:**
- `quarantine suite list`: discover repo root, read config, print each suite's
  name, command, and junitxml; exit 0
- `quarantine suite remove <name>`: read config, print ramifications, ask
  `[y/N]`, remove suite entry from config; MUST NOT delete state file on branch;
  MUST NOT delete or close associated issues
- `quarantine status [suite-name]`: read state + recent artifacts; show
  quarantined count, estimated CI time (from `duration_ms`), oldest quarantined
  tests; omit duration line when no artifacts available
- `timeout` (default 30m) and `rerun_timeout` (default 5m) config fields with
  Go duration format; `quarantine doctor` validates; SIGTERM then 5s SIGKILL;
  partial XML processed on command timeout; rerun timeout → `"unresolved"`
- `.quarantine/<suite>/quarantined-files.txt`: deduplicated quarantined file
  paths written before command execution; empty file when no quarantined tests
- Error prefix convention for all exit-2 messages: `Error [config]:`,
  `Error [crash]:`, `Error [timeout]:`, `Error [rerun]:`
- Dashboard: list `.quarantine/` on state branch (1 call); read per-suite
  `state.json` (N calls); handle 404 on `.quarantine/` gracefully
- Schema updates: `schemas/test-result.schema.json` — add `suite_name`, remove
  `framework`, add `"unresolved"` status, add `error`, add `rerun_exit_code`;
  `schemas/quarantine-state.schema.json` — per-suite path semantics
  (`last_flaky_at` → `last_failure_at` already done in M9)

**MUST NOT implement:**
- `quarantine suite add` — deferred; users edit YAML directly
- Combined PR comment aggregation — deferred to v2
- Per-framework exclusion adapters — deferred to v2
- `quarantine state consolidate` — deferred to v2
- Any features beyond M9's stable surface

## Acceptance criteria

From [index.md](index.md#m10-additive-features):

1. `quarantine suite list` prints each suite's name, command, and junitxml.
   (FR-1.4.2)
2. `quarantine suite remove backend` prints ramifications, asks `[y/N]`,
   removes suite from config; state file on branch preserved. (FR-1.4.2)
3. `quarantine status backend` shows quarantined count, estimated CI time,
   oldest tests with age and issue numbers. (FR-1.1.1)
4. `quarantine status` with no arg and multiple suites shows summary for all.
   (FR-1.1.1)
5. `timeout: 30m` enforced: SIGTERM at 30m, SIGKILL after 5s grace; partial
   JUnit XML processed; exit 2 with `Error [timeout]: ...`. (ADR-031)
6. `rerun_timeout: 5m` enforced: hanging rerun killed; test classified
   `"unresolved"`; run continues. (ADR-031)
7. `.quarantine/<suite>/quarantined-files.txt` written before command;
   deduplicated; empty when no quarantined tests. (plan D5)
8. All exit-2 messages use `Error [config]:`, `Error [crash]:`,
   `Error [timeout]:`, or `Error [rerun]:` prefix. (ADR-031)
9. Dashboard enumerates `.quarantine/` on state branch; reads per-suite
   `state.json`; handles 404 gracefully. (ADR-032, FR-1.5.2)
10. Results JSON has `suite_name`, no `framework`, `"unresolved"` status,
    `error` and `rerun_exit_code` on unresolved tests. (ADR-031)
11. Schema validation passes: `make contract-test`. (NFR-2.6.3)
12. `make test-all` passes (cli-test + dash-test + contract-test + e2e-test).
    (NFR-2.5.1)

## Verification

**Build:** `make test-all` MUST pass (cli-test + dash-test + contract-test +
e2e-test).

**Timeout enforcement:** MUST send SIGTERM first, wait 5 seconds, then SIGKILL.
MUST process partial JUnit XML if present at timeout. MUST exit 2 with
`Error [timeout]: test command timed out after {duration}.`

**Rerun timeout:** MUST kill rerun process and classify test as `"unresolved"`.
MUST continue processing remaining failed tests (MUST NOT abort run).

**Error prefix:** Every exit-2 error message MUST begin with one of:
`Error [config]:`, `Error [crash]:`, `Error [timeout]:`, `Error [rerun]:`.

**quarantined-files.txt:** MUST be written before the suite command executes.
MUST be empty (not missing) when no tests are quarantined.

**Dashboard state enumeration:** MUST handle `GET .quarantine/` returning
404 gracefully (no state to display, not an error). MUST NOT require
`.quarantine/` to exist before the first suite run.

**Schema:** `make contract-test` MUST pass with updated schema files.

**Requirements:** Implementation MUST satisfy [FR-1.4.2], [FR-1.5.2] in
[functional-requirements.md](../specs/functional-requirements.md)
and [NFR-2.5.1], [NFR-2.6.3] in
[non-functional-requirements.md](../specs/non-functional-requirements.md).
